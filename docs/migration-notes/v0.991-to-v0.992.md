# Migration notes: v0.991 → v0.992

This file documents FSM evolution. Source of truth is
`theories/fsm.md` and `core/check/verify.sh`.

## Lifecycle FSM: 6 states → 4 states

Earlier designs (`sketch / working / verified / deprecated /
archived / superseded`) were theoretical and never matched
the verifier's accepted enum. v0.992 collapses to
`draft / applied / retired / abandoned`, the FSM the code
actually implements.

Mapping for existing packets:

| Old label      | New label |
|----------------|-----------|
| `sketch`       | `draft`   |
| `working`      | `draft`   |
| `verified`     | `applied` |
| `deprecated`   | `retired` |
| `archived`     | `archived` directory (out of FSM) |
| `superseded`   | `retired` + `supersession: math/<successor>/` |

If you have packets with the old enum, edit
`packet.yaml` accordingly and re-run `sh math-coding verify`.

## verified_by field (v0.992)

`applied` packets now require `verified_by: [name, ...]`
listing peer reviewers. axiom packets are exempt.
`single_author: true` is required if there is only one
verified_by entry.
