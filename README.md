# math-coding v0.992

> Curry-Howard convention for AI coding agents.
> Plain-text. git. POSIX. Seven axioms. axiom Self-Application.

## What this is

A convention where every non-trivial decision is a **packet**
— a directory with three mandatory files and two auto-generated
files. The packet is the proposition; the verifier is the
type-check; the convention applies to itself (axiom Self-Application).

## Seven axioms

  A0. Difference       A4. Process
  A1. Care             A5. Accounting
  A2. Curry-Howard     A6. Self-Application
  A3. Material Basis

Read `core/spec/axioms.md` for the full statement.

## Packet

Mandatory:

  packet.yaml      manifest + lifecycle + applications[]
  decision.md      proposition (thesis / antithesis / synthesis)
  refinement.md    state / operation / invariant / test

Auto-generated by `create --from spec.yaml`:

  task.md          intent (problem / outcome / constraints)
  assumptions.yaml epistemic context (5 markers)

Each packet lives under `math/<name>/`.

## Three modes

  light    commit message only
  standard full packet (3 mandatory + 2 generated)
  strict   packet + theory link + applications[] + surface impact

## Lifecycle

Main line:

  draft → applied → retired

Side states:

  abandoned   (draft packet that will not be applied)
  archived    (retired packet moved to `math/archived/`)
  superseded  (replaced by a new packet)

`applied` requires at least one SHA witness in
`packet.yaml:applications[]` and at least one approving review.

## Five epistemic markers

  fact / hypothesis / judgment / unknown / proven

`proven` is reserved for axiom Self-Application.

## Five verdict outcomes

  VERIFIED, NEEDS_REVISION, UNVERIFIABLE:{TOOL_MISSING,
  DEFERRED, OUT_OF_SCOPE}.

## Commands

  sh math-coding create <name> --from <spec.yaml>
                              Create a packet from a 7-field spec.
  sh math-coding extract <name>
                              Emit a YAML spec for an existing packet.
  sh math-coding apply <name> [--tests=<cmd>] [--tests-result=<PASS|FAIL>]
                              Record SHA witness and transition to applied.
  sh math-coding review <name> --approve | --request-changes | --comment
                              Peer-review a packet.
  sh math-coding retire <name> --reason=<supersession|deprecation>
                              Transition to retired.
  sh math-coding abandon <name> [--reason=<text>]
                              Transition draft → abandoned.
  sh math-coding lifecycle <name> <applied|retired|abandoned>
                              Unified lifecycle transition.
  sh math-coding stable <name> [--unmark]
                              Mark or unmark packet as stable.
  sh math-coding archive <name> [--confirm]
                              Move retired packet to `math/archived/`.
  sh math-coding verify       Run structural checks.
  sh math-coding drift-check  Check applications[] SHA vs HEAD.
  sh math-coding probe        Run axiom Self-Application checks.
  sh math-coding install <path> [--gitignore]
                              Install into a brownfield project.
  sh math-coding upgrade <path>
                              Upgrade an existing install.
  sh math-coding uninstall <path>
                              Remove install from path.
  sh math-coding install-skill
                              Install the math-coding agent skill.
  sh math-coding config       Interactive `.mathrc` editor.
  sh math-coding help         Show usage.

## Quick start

  git clone math-coding-v0.992
  cd math-coding-v0.992
  sh math-coding probe         # axiom Self-Application
  cat > spec.yaml <<'YAML'
proposition: |
  My first decision.
outcome: |
  The codebase behaves as claimed.
invariant: |
  Existing tests still pass.
test: |
  Run the project's test suite.
antithesis: |
  The change could break existing callers.
synthesis: |
  Add tests before changing behavior; roll back if red.
operation: |
  Implement the change and run tests.
YAML
  sh math-coding create my-feature --from spec.yaml
  # edit the generated files under math/my-feature/
  git add math/my-feature spec.yaml
  git commit -m "my-feature: draft packet"
  sh math-coding verify

## Install in an existing project

  sh /path/to/math-coding/math-coding install /path/to/project
  cd /path/to/project
  sh ./.math-coding/math-coding probe
