Why

def add(a, b) is correct in arithmetic and wrong in string concatenation. The proposition must come from somewhere, and that somewhere is decision.md, separate from the code. Read more →

What it is, what it is not

What it is

  • A convention for AI coding agents that documents decisions as proof terms.
  • A formal grammar of decision-making, not a workflow for committing code.
  • Plain-text: .md, .yaml, .sh. Always.

What it is not

  • Not a framework. Not a test runner. Not a governance tool for non-AI commits.
  • Not a substitute for human review on adversarial work.
  • Not magic. It is a protocol for honest agents, not a shield against dishonest ones.

When not to use

  • Typos, renames, format fixes, dep bumps within minor version.
  • Throwaway scripts and demo codebases.
  • If you cannot commit the decision-grammar to your repository.

Seven axioms

Each axiom is a formal proposition. Each lives as a packet under math/<NN-name>/. Read them in order:

  1. A0

    Difference

    A proposition differs from its implementation. Without this gap, no convention is needed.

    ontological
  2. A1

    Care

    A developer cares whether the code does what it claims. Without care, no amount of structure helps.

    motivational
  3. A2

    Curry-Howard

    A packet is a proof term. A verifier is a type-check.

    structural
  4. A3

    Material Basis

    The convention lives in plain text, in git, and runs on POSIX shell.

    substrate
  5. A4

    Process

    Process precedes code. The lifecycle of a packet is finite and ordered.

    temporal
  6. A5

    Accounting

    Knowledge must be marked. Verdicts must be named. Changes must be witnessed.

    epistemic
  7. A6

    Self-Application

    The convention applies to itself.

    meta
