A convention where every non-trivial decision is a proof term — five plain-text files carrying the proposition, the antithesis, and the synthesis. The verifier is a type-check. The convention applies to itself.

Why

def add(a, b) is correct in arithmetic and wrong in string concatenation. The proposition must come from somewhere, and that somewhere is decision.md, separate from the code.

What it is, what it is not

What it is

  • A convention for AI coding agents that documents decisions as proof terms.
  • A formal grammar of decision-making, not a workflow for committing code.
  • Plain-text: .md, .yaml, .sh. Always.

What it is not

  • Not a framework. Not a test runner. Not a governance tool for non-AI commits.
  • Not a substitute for human review on adversarial work.
  • Not magic. It is a protocol for honest agents, not a shield against dishonest ones.

When not to use

  • Typos, renames, format fixes, dep bumps within minor version.
  • Throwaway scripts and demo codebases.
  • If you cannot commit the decision-grammar to your repository.
