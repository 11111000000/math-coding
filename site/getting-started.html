Get started

Prerequisites

math-coding requires:

  • A POSIX shell (/bin/sh, dash, ash, busybox sh — bash works but is not required).
  • git for SHA witnesses.
  • A text editor.

No Node, no Python, no JVM, no package manager. See axiom A3.

1. Probe the source repo

Before installing, verify the convention is internally consistent:

git clone https://github.com/11111000000/math-coding.git
cd math-coding
sh math-coding probe

This runs 51+ self-tests (axiom A6 in action). If probe exits 0, the convention holds at this commit.

2. Install in a project

Install into a brownfield project:

sh /path/to/math-coding/math-coding install /path/to/your-project

By default .math-coding/ is committed to the project. Use --gitignore to opt in to gitignored behavior.

3. Create your first packet

Write the proposition first. A worked example — cache TTL: 

cat > spec.yaml <<'YAML'
proposition: |
  Cache entries expire after 60 seconds, not at user request.
outcome: |
  Cache returns fresh entry within 60s of last refresh.
invariant: |
  Cache entries never served beyond TTL.
test: |
  Insert entry with ts = now - 61s. Read. Expect upstream fetch.
antithesis: |
  Manual invalidation may be needed; fixed TTL forces users to wait 60s.
synthesis: |
  TTL is fixed at 60s; manual invalidation is a separate endpoint.
operation: |
  On read, check entry timestamp. If age > 60s, refresh from upstream.
YAML

sh math-coding create cache-ttl --from spec.yaml

The convention creates math/cache-ttl/ with five files. Edit them; commit; apply.

4. Apply, review, verify

git add math/cache-ttl spec.yaml
git commit -m "cache-ttl: apply 60s expiration"
sh math-coding apply cache-ttl --sha=<commit-sha>
sh math-coding review cache-ttl --approve --note="60s is correct for /cache"
sh math-coding verify

The apply command records the SHA witness (axiom A5). The review command records peer approval. The verify command runs the structural check.

5. The five files

packet.yaml

type signature — manifest, lifecycle, reviews[]

decision.md

the proposition — Thesis / Antithesis / Synthesis

task.md

assumptions.yaml

context Γ — five epistemic markers, each with evidence

refinement.md
